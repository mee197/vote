# 「律·绘」商法导论海报设计展 - 投票系统

这是一个完全静态的HTML投票页面，无需服务器和数据库，适合分享给学生进行海报投票。

## 功能特点

- ✅ **多选投票**：每位评委可投15票
- ✅ **图片预览**：点击海报可放大查看详情
- ✅ **实时计数**：顶部显示已选数量/15
- ✅ **响应式设计**：适配手机、平板、电脑
- ✅ **绿色选中标记**：选中作品显示绿色边框和勾选图标
- ✅ **投票结果导出**：提交后自动下载JSON格式投票结果
- ✅ **无需登录**：打开链接即可投票

## 使用方法

### 1. 本地测试
直接双击 `index.html` 在浏览器中打开，或使用以下命令启动本地服务器：

```bash
# Python 3
python -m http.server 8000

# 或使用 npx
npx serve .
```

然后在浏览器访问：http://localhost:8000

### 2. 在线部署（推荐）

#### 方案一：GitHub Pages（完全免费）
1. 在GitHub创建新仓库
2. 将 `vote` 文件夹所有内容上传
3. 进入仓库 Settings → Pages
4. 选择 `main` 分支，保存
5. 几分钟后获得类似 `https://用户名.github.io/仓库名` 的链接

#### 方案二：Vercel / Netlify（免费）
- 拖拽 `vote` 文件夹到 Vercel/Netlify 网站
- 自动部署并获得 `xxx.vercel.app` 或 `xxx.netlify.app` 链接

#### 方案三：腾讯云COS/阿里云OSS（低成本）
- 上传到对象存储，开启静态网站托管
- 绑定自定义域名（可选）

## 投票规则说明

页面已内置以下规则说明：

1. **投票方式**：多选，每位评委可投 **15 票**
2. **操作方式**：点击海报卡片即可选中/取消，选中的海报会显示绿色边框和勾选图标
3. **查看详情**：点击海报图片可放大查看详情
4. **提交投票**：浏览完所有作品后，在页面底部查看已选数量并提交
5. **展览安排**：优秀作品将在教学楼走廊公开展出

## 技术说明

- 纯前端实现：HTML + CSS + JavaScript
- 无后端依赖：所有逻辑在浏览器中运行
- 数据持久化：投票结果以JSON文件形式下载到本地
- 图片懒加载：提升页面加载性能
- 响应式设计：适配各种屏幕尺寸

## 自定义修改

如需修改投票规则（如票数限制），编辑 `index.html` 中的以下部分：

```javascript
// 第 ~120 行
const MAX_VOTES = 15; // 修改此处数字

// 第 ~150 行
<li><strong>投票方式：</strong>多选，每位评委可投 <strong>15 票</strong></li>
```

## 注意事项

1. 投票结果为**本地存储**，如需集中收集，请让学生将生成的JSON文件发送给组织者
2. 图片文件较大时，首次加载可能较慢，建议部署在CDN
3. 如需防止重复投票，需要后端支持（当前版本无此功能）
4. 建议在投票前向学生说明规则和操作方法
